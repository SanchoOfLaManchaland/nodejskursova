## ВМІСТ ЗАПИСКИ

### 1. ВСТУПНА ЧАСТИНА

Веб-додаток "D&D Character Manager" розроблений для полегшення створення та керування персонажами в настільних рольових іграх Dungeons & Dragons. Система дозволяє користувачам реєструватися, створювати персонажів з повною характеристикою (статура, броня, хіти), редагувати та видаляти їх, а також зберігати додаткову інформацію (способності, інвентар, замітки) в безпечній базі даних MongoDB.

### 2. МЕТО ЗАДАЧІ

Розробити повнофункціональний веб-додаток з наступними можливостями:
- Автентифікація та авторизація користувачів (bcryptjs для хешування)
- CRUD операції для персонажів (створення, перегляд, редагування, видалення)
- Зберігання даних у MongoDB з Mongoose ODM
- TypeScript з strict режимом
- Юніт-тести (24 тести, покриття роутів та middleware)
- CI/CD з лінтингом та тестуванням (GitHub Actions)

### 3. ТЕХНОЛОГІЧНИЙ СТЕK

**Backend:**
- Node.js + Express.js
- TypeScript (strict mode, ts-jest для тестів)
- MongoDB + Mongoose ODM
- express-session для управління сесіями
- bcryptjs для хешування паролів

**Frontend:**
- EJS шаблонизатор
- CSS з адаптивністю
- Респонсивний дизайн

**DevOps:**
- GitHub Actions CI/CD
- Jest + Supertest для інтеграційних тестів
- ESLint 9+ з TypeScript ESLint

### 5. ОПИС ФУНКЦІОНАЛУ

#### 5.1. Головна сторінка (публічна)
- Опис можливостей додатку
- Навігація до реєстрації/авторизації
- Hero-секція з призивом до дії

#### 5.2. Сторінки автентифікації (2 сторінки)
- Реєстрація нового користувача (username, email, password)
- Вхід існуючого користувача
- Хешування паролів через bcryptjs
- Управління сесіями через express-session

#### 5.3. Сторінки персонажів (з аутентифікацією)
- Список усіх персонажів користувача (грид)
- Перегляд деталей персонажа (таблиця характеристик)
- Створення нового персонажа (форма з полями)
- Редагування існуючого персонажа
- Видалення персонажа

#### 5.4. Динамічний сегмент
- Динамічне відображення списку персонажів
- Форми створення/редагування з валідацією
- Перенаправлення після дій

### 6. БАЗА ДАНИХ

**Колекція users:**
- username (унікальний, 3-30 символів)
- email (унікальний)
- password (захешований через bcrypt)
- characters (масив ObjectId)

**Колекція characters:**
- name, race, characterClass (раси та класи D&D)
- level (1-20)
- background (Оріджін персонажа)
- stats (вбудована схема: strength, dexterity, constitution, intelligence, wisdom, charisma)
- armorClass, hitPoints, speed
- abilities, inventory, notes (додаткова інформація)
- owner (посилання на User)

### 7. ТЕСТУВАННЯ

Автоматичні тести (24 тести):
- Маршрути автентифікації (GET login/register)
- Перевірку доступу до захищених ресурсів
- Тести middleware аутентифікації
- Інтеграційні тести роутів

Для запуску: `npm test`

### 8. CI/CD

- Лінтинг коду (ESLint)
- Запуск тестів з покриттям
- Збереження артефактів

### 9. UI/UX ОПТИМІЗАЦІЯ

- Адаптивний дизайн (mobile-first, media queries)
- Інтуїтивна навігація
- Грид-раут для персонажів
- Кнопки дій (створити, редагувати, видалити)

### 10. ВИКЛИКИ ТА РІШЕННЯ

**Виклик 1:** TypeScript strict mode + Mongoose
**Рішення:** Використання `as any` для доступу до сесії через додавання декларації модуля

**Виклик 2:** Покриття коду 40%
**Рішення:** Створено 24 інтеграційних тесту (роути та middleware), які покривають основну логіку

**Виклик 3:** Безпека паролів
**Рішення:** Хешування bcryptjs з авто-генерацією salt

**Виклик 4:** Mongoose + TypeScript типи
**Рішення:** Використання generics для моделей, визначення інтерфейсів
